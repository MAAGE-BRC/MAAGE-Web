Bacterial Outbreak Investigation

The Bacterial Outbreak Investigation workflow supports genome assembly, annotation, genomic surveillance, cluster detection, comparative genomics, phylogenetic analysis, antimicrobial resistance characterization, and epidemiological interpretation within a unified workspace and analysis environment. Designed for routine surveillance and outbreak response, it enables rapid comparison of new isolates with public genomic databases to identify related cases, characterize transmission clusters, and support evidence-based public health investigations.

Step 1. Upload Your Sequencing Data

Begin by uploading one or more samples to your investigation.

MAAGE supports multiple starting points, allowing you to begin with:

  • • Raw sequencing reads (FASTQ)
  • • Assembled genome contigs (FASTA)
  • • Previously analyzed genomes in your MAAGE workspace

Each uploaded sample becomes part of your investigation and can be analyzed individually or together throughout the workflow.

Supported inputs

  • • Illumina paired-end FASTQ
  • • Oxford Nanopore FASTQ
  • • Assembled genome contigs (FASTA)
  • • Existing MAAGE genomes

Next Step

Run Comprehensive Genome Analysis to assemble (if needed), annotate, and characterize each genome.

Step 2. Identify and Characterize Your Pathogen

Comprehensive Genome Analysis

Comprehensive Genome Analysis performs genome assembly (when needed), genome annotation, and generates a comprehensive summary of each organism, including species identification, typing information, antimicrobial resistance genes, virulence factors, and other key genomic characteristics.

For supported organisms, MAAGE automatically performs:

  • • cgMLST allele calling using ChewBBACA
  • • HierCC hierarchical clustering
  • • Comparison against the public genome database

These analyses immediately identify closely related public isolates and provide the first indication of potential outbreak connections.

Outcome

  • • Annotated genome
  • • Species identification
  • • MLST and serotype (when available)
  • • AMR gene predictions
  • • Virulence factors
  • • cgMLST profile
  • • HierCC cluster assignments
  • • Closest related public genomes

Step 3. Review Related Public Isolates

Once your genome has been characterized, examine its relationship to publicly available genomes.

Review the automatically assigned HierCC clusters and explore closely related public isolates to determine whether your sample belongs to a known lineage, previously reported outbreak, or ongoing surveillance cluster.

Use this information to assess:

  • • Geographic distribution
  • • Collection dates
  • • Host and source metadata
  • • Historical outbreak associations
  • • Potential epidemiological links

This step often provides the first evidence that multiple cases may be connected.

Step 4. Compare Suspected Outbreak Isolates with Core Genome MLST

If multiple isolates appear to be related, perform a focused Core Genome MLST analysis.

Unlike the automatic HierCC comparison performed during Comprehensive Genome Analysis, this service compares only the genomes you select, allowing you to analyze combinations of private and public genomes or exclusively private datasets.

Core Genome MLST identifies shared and differing alleles among your selected genomes and produces an interactive minimum spanning tree to visualize genetic relationships.

Use this analysis to determine:

  • • Which isolates belong to the same outbreak
  • • Which isolates are unrelated
  • • Gene-by-gene differences
  • • Genetic distance between isolates

Outputs

  • • Allele difference matrix
  • • Cluster assignments
  • • Minimum spanning tree (GrapeTree MSTree V2)
  • • Pairwise genome comparisons

Step 5. Resolve Closely Related Isolates with Whole Genome SNP Analysis

When isolates remain highly similar after cgMLST analysis, Whole Genome SNP Analysis provides higher-resolution comparisons.

Using kSNP4, MAAGE identifies:

  • • Core SNPs
  • • Majority SNPs
  • • All SNPs

Pairwise SNP distances, SNP heatmaps, and phylogenetic trees help distinguish nearly identical isolates and infer transmission events with greater confidence.

Use this analysis to:

  • • Confirm outbreak membership
  • • Differentiate highly similar isolates
  • • Identify transmission pathways
  • • Measure evolutionary distance

Outputs

  • • SNP matrices
  • • Pairwise SNP distances
  • • SNP heatmaps
  • • Maximum Likelihood, Neighbor Joining, and Parsimony trees

Step 6. Visualize Evolutionary Relationships

Construct phylogenetic trees to summarize the evolutionary relationships among your investigation samples.

The Phylogenetic Tree service builds codon-based phylogenies and allows metadata to be mapped directly onto the tree, making it easier to interpret transmission patterns, geographic spread, temporal trends, and outbreak structure.

Although Core Genome MLST and Whole Genome SNP Analysis also generate phylogenetic trees automatically, this service provides greater flexibility for building customized trees from selected genomes.

Use this step to:

  • • Visualize transmission networks
  • • Explore evolutionary relationships
  • • Generate publication-quality figures
  • • Communicate investigation findings
